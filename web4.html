Formátování HTML

Každý text má obsah a formu. Když mluvím o formátu (formě) webových stránek, myslím tím třeba barvu a velikost písma, pozadí, zarovnání atd., prostě všechno, co nepatří do obsahu. To je formátování.

Protože se jazyk HTML vyvíjel, vznikaly časem různé způsoby, jak formátovat text. Proto dnes existují dva odlišné způsoby, jak v HTML třeba obarvit písmo nebo ztučnit text.

  1. Starší způsob používá přímo HTML tagy. (Například kurzíva se dělá pomocí tagů <i> a </i>: <i>kurzíva</i>). Pomocí tagů se však některé věci nedají udělat.
  2. Novější způsob -- CSS styly -- používá tag <style> a obecný atribut "style", kterému se přiřazuje nějaká definice. Je to složitější, ale užitečnější a všeobecné.

Co je CSS?

CSS vzniklo někdy kolem roku 1997 v Norsku. Je to kolekce metod pro grafickou úpravu webových stránek. Ta zkratka znamená Cascading Style Sheets, česky "kaskádové styly". Kaskádové, protože se na sebe mohou vrstvit definice stylu, ale platí jenom ta poslední. Poslední verzí kaskádových stylů je CSS3, které do možností přidává třetí rozměr, práci s vektorovou grafikou, animace, nové typy selektorů, nové vlastnosti pozadí, nové vlastnosti rámečku - což je důležité především proto, že se díky tomu dají v CSS dělat různé tvary (od čtverce po hvězdu), nastavení sloupců (jejich šířky, mezer mezi nimi..) atd. V kombinaci s nejnovější verzí HTML5 tvoří základ moderních webů.

Výhody používaní kaskádových stylů:
  1. Rozsáhlejší možnosti formátování
  2. Jednodušší údržba webové prezentace (pokud chceme změnit např. barvu nadpisu, nemusíme složitě hledat v HTML dokumentu, ale stačí změnit jednu vlastnost v CSS souboru).
  3. Oddělení struktury a stylu (v jednom dokumentu budeme mít pouze sémanticky označen obsah a v druhém dokumentu máme definice vzhledu stránek).
  4. Cachování stylů – styly můžou být uložený do cache paměti, tím můžeme docílit zrychlení načtení stránek. Na druhou stranu při použití externího CSS souboru dochází k dalšímu http požadavku.
  5. CSS vlastnosti jednotlivých elementů můžeme dynamicky měnit pomocí Javascriptu.
  6. Pomocí CSS můžeme formátovat jakýkoliv XML jazyk.
  7. Můžou existovat různé styly pro různá zařízení (jiný styl proč počítač, pro mobil, pro tisk).
  8. Koncový uživatel může napsat vlastní CSS styl pro libovolnou stránku (většina prohlížečů podporuje uživatelské styly, takže si může uživatel nastavit, aby se mu např. hypertextové odkazy zobrazovaly jinou barvou a podtrženě).
  9. V kombinaci s Javascriptem může velmi vylepšit vzhled stránky.
Nevýhody používání kaskádových stylů:

Ne vždy dostatečná podpora v majoritních prohlížečích (prohlížeče obsahují v implementaci CSS chyby a je někdy nesnadné kód napsat tak, aby se ve všech běžných prohlížečích výsledek zobrazil stejně).

Trojí použití CSS

Styl se může nadeklarovat třemi způsoby, níže uvádím příklady. Stačí, když se pro začátek naučíte naučíte jeden ze tří způsobů:
  1. Přímý styl - Přímo v textu zdroje u formátovaného elementu pomocí atributu style="...". <p style="color: red">Tento odstavec bude červený.</p>
  2. Stylopis (angl. "stylesheet") - v hlavičce stránky. Stylopis je jakýsi seznam stylů. Je v něm obecně napsáno, co má být jak zformátováno, například že nadpisy mají být zelené. Do stránky se stylopis píše mezi tagy <style> a </style>. <style>p {color: red}</style>
  3. Použitím externího stylopisu -- to je soubor *.css, na který se stránka odkazuje tagem <link>. V souboru je umístěný stylopis. Hlavní výhoda je v tom, že na jeden takový soubor se dá nalinkovat mnoho stránek, takže pak všechny vypadají podobně. <link rel="stylesheet" type="text/css" href="styly.css">

CSS3

Jak jsem již zmínil v úvodu, nejnovější verzí kaskádových stylů je verze s číslovkou 3 na konci. Ta přináší mnoho novinek a s nimi však spojeno spoustu problémů - především s podporou jednotlivých prohlížečů. V současné době totiž každý prohlížeč vyžaduje vlastní prefix u některch atributů, což práci programátora značně ztěžuje a prodlužuje. Od roku 2011, kdy se poprvé objevily “moduly třetí třídy” se tato situace změnila k lepšímu, ovšem k větší standardizaci těchto atributů zatím nedošlo. CSS3 zavedlo do světa kaskádových stylů nový pojem - moduly. Jazyk už se nevytváří kompletní, ale tvoří se pro něj jen nové metody a zápisy (moduly) které se později implementují. Proto můžeme už nyní slyšet o “CSS4”, které vlastně nepředstavuje nic jiného, než moduly čtvrté kategorie pro CSS3.

Animace

Jednou z novinek v CSS3 je možnost jednoduchého animování elementů webu. Tyto animace samozřejmě nedosahují velikosti například jQuery, ovšem i tak se s nimi dá zahrát velké divadlo. Animace v kaskádových stylech užívají několika způsobů svého vytvoření.

Transitions - přechody

Transition vlastnosti zapisujeme přímo do css definice jednotlivých elementů, pro které chceme animaci použít.
  • První vlastnost transition-property určuje, jakou vlastnost budeme animovat. Hodnota tedy může být třeba color, border. Více vlastností, které chceme animovat, oddělujeme čárkou. transition-property: width, border, left;.
  • Vlastností transition-duration definujeme délku animace v sekundách. Tedy například 0.5s, 2s apod...transition-duration: 0.8s;.
  • Poslední vlastností je transition-timing-function, která udává, jaká funkce se má použít na výpočet průběhu animace. Možnosti jsou: ease (default) - bez hodnoty, linear, ease-in, ease-out, ease-in-out,step-start, step-end, lze definovat i vlastní, pomocí hodnoty cubic-bezier() pro průběh podle námi zadané Bézierovy křivky, nebo steps() pro zadání kroků.
Transitions animace se dají použít například pro prolínání dvou elementů, “načtení” některého elementu apod.

Keyframes

Keyframes animace umožňuje nadefinovat klíčové body a k těmto bodům přiřazovat, co všechno animace v daný bod udělá. Jak jsem již zmínil v úvodu, tento typ animace umožňuje pojmenování a následnou aplikaci na libovolný počet prvků. To velmi zkrátí zápis u větších projektů. Jsme ale opět nuceni provádět zápis vlastnosti pro každý prohlížeč zvlášť. Bez prefixu, -webkit-, -o- a -moz-. Samotnou vlastnost zapisujeme zavináčem, názvem vlastnosti a námi zvoleným názvem dané animace. @keyframes nase_animace. Obsah vlastnosti opět zapisujeme do složených závorek {}.
